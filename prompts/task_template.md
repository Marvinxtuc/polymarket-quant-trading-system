# 阻断项任务模板（Task Template）

必填
- BLOCK-ID：
- 摘要（1 行）：
- 完成定义：代码 / 静态门 / 测试 / 验证脚本 / 回归 / 文档 / 自检

约束
- 不得改动非本 BLOCK 的业务逻辑。
- 不得删除/弱化测试；失败默认阻断。
- 依赖项需最小化、显式列出。

交付物
- 代码改动清单
- 测试与验证命令 + 结果
- 文档与报告路径
- 自检报告

执行步骤
1) 方案设计：范围、风险、回滚。
2) 实施：按子任务逐项完成。
3) 门禁：static / tests / docs / block-item 全部执行。
4) 报告：输出到 `reports/blocking/<BLOCK-ID>/`.
