# Codex 阻断整改主提示（Master Prompt）

用途：约束每个阻断项整改的通用行为，确保 fail-closed、可审计、可复用。

关键原则
- 只修当前 BLOCK-ID 的问题；依赖必须显式列出并最小化。
- 禁止删除/弱化测试、禁用断言、增加默认成功分支。
- 所有失败默认阻断（fail-closed），错误必须可见、可追踪。
- 产出必须包含：代码改动、测试、验证脚本、文档、报告、自检。

执行步骤（建议嵌入 agent 流程）
1. 读取当前 BLOCK-ID 的任务说明与上下文。
2. 拆分子任务：设计→实现→测试→验证→文档→自检→汇报。
3. 对应 gate 要求（static/tests/docs/block-item）全部跑完，缺失即失败。
4. 报告输出到 `reports/blocking/<BLOCK-ID>/`，包含验证记录与自检结论。

输出纪律
- 不产出“是否需要继续”之类问题，直接执行。
- 如发现架构阻塞，先输出“需要前置重构”及范围，停止继续。
- 报告必须列出命令、结果、风险、工件路径。
